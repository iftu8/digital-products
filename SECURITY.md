# 🛡️ Enterprise Security Policy

Welcome to the official security policy for our Digital Products and AI Ecosystem. We take the security, data integrity, and privacy of our systems extremely seriously.

## 🟢 Supported Versions

Below is a list of product versions that are currently being monitored and supported with active security patches. We recommend all users upgrade to the latest stable release.

| Engine / Product Version | Security Support   |
| ------------------------ | ------------------ |
| Version 3.x (Latest)     | :white_check_mark: |
| Version 2.5.x            | :white_check_mark: |
| Version 1.0.x            | :x: (Deprecated)   |
| Beta / Alpha Builds      | :x:                |

## 🚨 Reporting a Vulnerability

If you are a security researcher or user and you have discovered a critical security vulnerability within our core engines, API connectors, or any digital product, please **DO NOT** open a public GitHub issue.

Please follow our responsible disclosure process:

1. **Direct Contact:** Send an email directly to **[এখানে আপনার ইমেইল এড্রেস দিন]**.
2. **Subject Line:** Use the subject line: `[URGENT: SECURITY VULNERABILITY] - <Product Name>`
3. **Details Required:** Please include detailed steps to reproduce the vulnerability, the potential impact, and any relevant scripts or screenshots.

### ⏱️ Incident Response SLA
* **Acknowledgement:** We will review and acknowledge your report within **24 hours**.
* **Action & Patch:** A patch or mitigation strategy will be developed within **72 hours** for high-severity issues.

Thank you for helping us keep our infrastructure bulletproof and our community safe!
